# DeepVision

## DETR (DEtection TRansformers)

DETR is a Transformer for Objects Detection in Image. Its Primary work is to predict the objects in the image and their position in the image. Based on the Position and size of the object we can draw  a bounding box. It takes an image as input and gives predictions for 100 objects in the image. Each prediction has 92 classes (including No object class) and their probabilities associated with it. One with highest probability above threshold is assumed tobe the correct class.


## Encode-Decoder Architecture

Encoder-decoder work together to identify objects in the image.

### Encoder 
Encoder uses self attention to understand the relation between different patches of the image and how closely are they related. Feature extraction happens based on which objects in the image to focus on. Key features of the image become prominent and the others become redundent. Intent of the Encoder is to expose each patch of the image to the other to underatand correlations and which information is imporatnt and has to be kept.

### Decoder
Decoder uses attention (NOT self attention), the queries come from different source which are called object queries. The intent of the query is to question any aspect of the image and get the results. If there are 100 questries that Decoder can do to an image, it will get 100 outputs, one for each object query. In DETR context, 100 Object queries are 100 bounding boxes and the intent is to extract 100 objects from the image, classify each of the 100 object into 92 classes and predict the Confidence score for each class. It also predicts the Bounding box location and size.


## Bipertite Loss

When we are training DETR, we need to check the Bounding box predictions it has done on the image and penalize for incorrect predictions. The way we do it is by using Bipertite Loss. Bipertite loss is basically a way of matching the Predictions with actual annotated image so that for ALL the Bounding Box predictions the Loss is minimum i.e. the Bounding Box mapping should be done in a way that the LOSS Value is Minimum. This way we can assume that each Bounding Boxe prediction has been correctly mapped to its closest match in the Actual annotated image. This will ensure that predicted Bounding Box mapping is done correctly with respect to Annotated image.

## OBJECT Queries

Decoder Object Queries is a way to Query an image to get the answer to the query. We can query an image to check whats in the image, we can also query an image to change the part of the image etc. In case of DETR, there are 100 Object queries, one responsible for a small part of image to check if there is an object in that pert of the image and predict the class with Bounding Box. Thats the reason we have 100 Ojects as output with each output having confidence score of 92 class predictions.
